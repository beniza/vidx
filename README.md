# VIDX — Scripture Video Generator
> **The official video rendering companion to [AUDX](https://github.com/bridgeconn/audx) (Audio Scripture Conversion Tool suite).**

`vidx` is a powerful, CLI-first, configuration-driven automation engine designed for Bible translation teams. It combines three core Bible translation assets into broadcast-quality lyric and scripture subtitle videos:
1. **USFM Scripture Files** (e.g., `42MRKsnd.SFM`)
2. **Audio Recordings** (e.g., `SND-MRK-5.mp3`, typically produced via **AUDX**)
3. **Verse-Level Timing Maps** (e.g., `C01-01-MRK-05-timing.txt` generated by Aeneas / Scripture App)

---

## 🏗️ System Architecture

`vidx` sits cleanly on top of our shared `usfm-converter` parser package, transforming static text and timing data into dynamically styled video streams:

```
┌────────────────────────────────────────────────────────┐
│                   VIDX Automation Engine               │
├──────────────────┬──────────────────┬──────────────────┤
│  Config Module   │  ASS Generator   │  FFmpeg Builder  │
│  (YAML / Rules)  │  (Styles/Events) │  (Filter/Render) │
└────────┬─────────┴────────┬─────────┴────────┬─────────┘
         │                  │                  │
         ▼                  ▼                  ▼
  [config.yaml]      [usfm-converter]      [FFmpeg + libass]
                    (USFM & Timing Map)
```

### Key Modules:
- **`usfm-converter` (Shared Core):** Robust USFM 3.0 syntax parser that removes formatting noise (footnotes `\f`, cross-references `\x`, and figure tags `\fig`) while aligning text segments with audio timestamps.
- **`ASSGenerator`:** Compiles USFM verses and timing entries into Advanced SubStation Alpha (`.ass`) subtitle files. Handles complex Indic/Devanagari text shaping, section headings (`\s1`, `\s2`), and customizable verse numbering (`5:1`).
- **`FFmpegBuilder`:** Dynamically builds hardware-accelerated FFmpeg command pipelines. Loops static image backgrounds or video loops, pads/crops to any aspect ratio, and composites subtitles via `libass` + `HarfBuzz`.
- **`BatchRunner`:** Orchestrates multi-chapter rendering jobs sequentially or in parallel across multi-core systems.

## 🌟 Key Features

| Feature | Description |
| :--- | :--- |
| **AUDX Companion** | Seamlessly consumes audio assets and timing maps generated during standard AUDX audio workflows. |
| **Multi-Aspect Support** | Render horizontal widescreen (`1920x1080`), vertical mobile shorts/reels (`1080x1920`), or social square (`1080x1080`). |
| **Complex Script Mastery** | Full support for complex scripts (Devanagari, Sindhi, Arabic, Thai) with ligatures, nuktas, and conjuncts via HarfBuzz text shaping. |
| **Smart Backgrounds** | Automatically loop short background video clips or animate static images to match audio duration. |
| **High-Contrast Styling** | Configurable semi-transparent background boxes behind subtitles for readability over dynamic backgrounds. |
| **Parallel Batching** | Process entire Bible books concurrently with multi-worker support (`-w 4`). |

## 🚀 Installation

### 1. Prerequisites
- **Python:** 3.7 or higher
- **FFmpeg:** Version 4.3+ compiled with `--enable-libass` and `--enable-libharfbuzz` (standard on modern Linux/Windows builds).
- **Fonts:** Target fonts (e.g., `Nirmala UI`, `Mangal`, `Noto Sans Devanagari`) must be installed on your operating system.

### 2. Package Setup
Install the underlying parser and the `vidx` engine in editable or standard mode:

```bash
# 1. Install usfm-converter package
cd path/to/usfm-converter
pip install -e .

# 2. Install vidx companion engine
cd path/to/vidx
pip install -e .
```

Verify the CLI installation:
```bash
vidx --help
```

## 🛠️ Command-Line Usage

### Mode 1: Configuration-Driven Batch Jobs (Recommended)

1. Create or copy a project configuration file:
   ```bash
   cp config.sample.yaml my_project.yaml
   ```

2. Run the rendering pipeline:
   ```bash
   # Execute all jobs listed in my_project.yaml
   vidx -c my_project.yaml

   # Execute with 4 parallel CPU workers for rapid processing
   vidx -c my_project.yaml -w 4

   # Shakedown Test: Render only the first 15 seconds of each job
   vidx -c my_project.yaml -t 15
   ```

### Mode 2: Quick Direct Conversion

Convert a single chapter directly without writing a YAML file:

```bash
vidx \
  --usfm src/42MRKsnd.SFM \
  --timing src/C01-01-MRK-05-timing.txt \
  --audio src/42MRKSND.mpeg \
  --bg src/bg.mp4 \
  -o output/mark_ch5.mp4
```

### Mode 3: Subtitle Generation Only

Compile standalone `.ass` subtitle files for manual inspection or editing in Aegisub:

```bash
vidx --generate-only \
  --usfm src/42MRKsnd.SFM \
  --timing src/C01-01-MRK-05-timing.txt \
  -o output/mark_ch5.ass
```

## 🎨 YAML Configuration Guide

Below is an overview of `config.yaml` customization options:

```yaml
project:
  name: "Sindhi Mark Video Project"
  output_dir: "output"

video:
  resolution: "1920x1080"       # 1920x1080 (Horizontal), 1080x1920 (Vertical Shorts), 1080x1080 (Square)
  fps: 24
  codec: "libx264"
  preset: "fast"
  crf: 23                       # Quality: 18 (near lossless) to 28 (compressed)
  background_media: "src/bg.mp4" # Video clip or static image (.jpg/.png)
  loop_background: true         # Loop background video to match audio length
  scaling_mode: "pad"           # Aspect ratio handling: 'pad' (black bars), 'crop', or 'stretch'

style:
  verse:
    font: "Nirmala UI"
    size: 48
    color: "#FFFFFF"            # Primary text color (#RRGGBB)
    outline_color: "#000000"
    outline_width: 3
    shadow: 1
    alignment: 2                # 2 = Bottom-Center (standard subtitle positioning)
    margin_bottom: 60
    margin_lr: 60
    background_box: true        # Semi-transparent bounding box for legibility
    background_color: "#00000080" # #RRGGBBAA (80 = 50% opacity)

  heading:
    font: "Nirmala UI"
    size: 56
    color: "#FFD400"            # Gold color for section titles (\s1, \s2)
    alignment: 8                # 8 = Top-Center
    margin_vertical: 80
    bold: true

  verse_number:
    show: true                  # Display reference prefix (e.g., "Mark 5:1")
    color: "#FFC080"            # Light orange/gold emphasis
    size: 36
    on_every_segment: false     # If false, displays reference only on first fragment (a) of a split verse

jobs:
  - usfm: "src/42MRKsnd.SFM"
    timing: "src/C01-01-MRK-05-timing.txt"
    audio: "src/42MRKSND.mpeg"
    output: "output/mark_ch5.mp4"
    background: "src/bg.mp4"
```
